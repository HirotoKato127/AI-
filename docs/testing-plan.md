# Agent Key 社内運用テスト計画

> 目的: 本番利用を開始する前に、Agent Key が問題なく動くことを確認する
> 対象: 3名（現テスター）→ 20名（全社展開）

---

## テストの全体像

```
Phase 1: 基本機能テスト（1人でやる）
  → 各画面が正しく動くか
    ↓
Phase 2: 業務シナリオテスト（2〜3人でやる）
  → 実際の業務フローを通しでやってみる
    ↓
Phase 3: 負荷・セキュリティ確認
  → 20人で使っても大丈夫か
    ↓
Phase 4: 全社展開
```

---

## Phase 1: 基本機能テスト

各画面を1つずつ開いて、操作が正しく動くか確認する。

### ログイン・認証

- [ ] ログインできるか
- [ ] ログアウトできるか
- [ ] 間違ったパスワードでエラーが出るか
- [ ] ログイン後にページを更新してもログイン状態が維持されるか

### 候補者管理（candidates）

- [ ] 候補者一覧が表示されるか
- [ ] 候補者を新規登録できるか
- [ ] 候補者の詳細を開けるか
- [ ] 候補者情報を編集・保存できるか
- [ ] 備考欄に文字を入力して保存できるか
- [ ] CSステータスを変更できるか
- [ ] CSステータス変更時にメール送信確認ダイアログが出るか
- [ ] 他社選考状況を保存できるか
- [ ] ネクストアクション（日付・メモ）が保存できるか

### テレアポ（teleapo）

- [ ] テレアポ一覧が表示されるか
- [ ] ダイヤルフォームが開けるか
- [ ] 通話記録を保存できるか
- [ ] SMSフォームが動くか
- [ ] CSタスクテーブルが表示されるか
- [ ] フィルター機能が動くか

### イールド / KPI（yield）

- [ ] 管理者ビューが表示されるか
- [ ] 個人ビューが表示されるか
- [ ] 企業別ビューが表示されるか
- [ ] 数値が正しく計算されているか（手動で検算）
- [ ] 期間フィルターが動くか

### その他の画面

- [ ] 目標設定画面が開けるか
- [ ] 紹介（リファラル）画面が動くか
- [ ] 設定画面が動くか
- [ ] マイページ/ダッシュボードが表示されるか
- [ ] 広告管理画面が動くか

---

## Phase 2: 業務シナリオテスト

実際の業務の流れを最初から最後まで通してやってみる。

### シナリオ1: 新規候補者の登録〜面接設定

```
1. 候補者を新規登録する
2. テレアポで電話をかける（ダイヤルフォーム）
3. 通話結果を記録する
4. 面接日程を設定する
5. ネクストアクションを設定する
→ イールド画面に反映されているか確認
```

- [ ] 一連の流れが問題なく動くか
- [ ] 途中でエラーが出ないか
- [ ] 入力したデータが他の画面にも正しく反映されるか

### シナリオ2: 選考プロセスの進行

```
1. 候補者の詳細を開く
2. 企業に推薦する（応募を追加）
3. 1次面接 → 2次面接 → 最終面接 → 内定のステータスを進める
4. 内定承諾を記録する
5. 入社日を記録する
→ イールドのKPIに正しく反映されるか確認
```

- [ ] 各ステータスの遷移が正しく動くか
- [ ] 日付が正しく保存されるか
- [ ] KPI計算結果が合っているか

### シナリオ3: CSステータスとメール送信

```
1. 候補者のCSステータスを「34歳以下メール(tech)」に変更
2. 確認ダイアログが出るか
3. OKすると自動メールが送信されるか
4. 「🎉 自動メールを送信しました！」のアラートが出るか
```

- [ ] メール送信機能が正しく動くか
- [ ] メールの内容が正しいか（テスト用アドレスで確認）

### シナリオ4: 複数人の同時操作

```
2〜3人が同時にログインして、
それぞれ別の候補者を操作する
```

- [ ] 同時にログインできるか
- [ ] 他の人の操作とデータが混ざらないか
- [ ] 同時保存でエラーが出ないか

## Phase 3: 非機能テスト

### パフォーマンス

- [ ] ページの読み込みが **3秒以内** か
- [ ] 候補者一覧（100人以上）の表示が遅くないか
- [ ] 検索・フィルターのレスポンスが適切か

### セキュリティ

- [ ] ログインしないとデータにアクセスできないか
- [ ] URLを直接入力してもログインページに飛ばされるか
- [ ] 他のユーザーのデータが見えてしまわないか（権限設定がある場合）

### ブラウザ互換性

- [ ] **Chrome**（最新版）で動くか
- [ ] **Safari** で動くか（Macユーザー向け）
- [ ] **Edge** で動くか（Windowsユーザー向け）
- [ ] **スマホ** でも最低限表示されるか

### エラーハンドリング

- [ ] ネットワーク切断時にエラーメッセージが出るか
- [ ] 必須項目を空にして保存した時にバリデーションが効くか
- [ ] 不正なデータを入力した時にクラッシュしないか

---

## Phase 4: 全社展開の準備

### テスト完了後にやること

- [ ] テストで見つかったバグをすべて修正
- [ ] 利用マニュアル（簡易版）を作成
- [ ] 社員全員のアカウントを作成
- [ ] ログインURL（`https://agent-key.pages.dev`）を全員に共有
- [ ] 問い合わせ先（バグ報告先）を決める

### 展開方法

```
おすすめ: 段階的に展開する

Week 1: テスターの3人だけ（今の状態）
Week 2: +5人追加して8人で使ってみる
Week 3: 問題なければ残り12人を追加して20人全員
```

---

## バグ報告テンプレート

テスト中にバグを見つけたら、以下の情報をメモしておく：

```
【画面名】 例: 候補者詳細
【操作内容】 例: 備考欄に入力して保存を押した
【期待する動作】 例: 保存されて画面に反映される
【実際の動作】 例: エラーが出て保存されなかった
【ブラウザ】 例: Chrome 最新版
【スクリーンショット】 あれば添付
```

| 期間 | Phase | やること | 担当 |
|---|---|---|---|
| 1週目 | Phase 1 | 基本機能テスト | 1人 |

| 3週目 | Phase 3 + バグ修正 | 非機能テスト＋修正 | 開発者 |
